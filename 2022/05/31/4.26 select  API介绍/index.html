0.1 select主旨思想


![[8fe3d70dbc83fb8ddb4ae30dad6893b8.png]]

  1. 创造一个文件描述表，将想要监听的函数(accept，read，rec)文件描述符添加进表
  2. 调用系统内核函数，监听该表，如果有函数进行了读写操作就对文件描述符表进行设置
  3. 返回文件描述表，并且告诉你哪些函数进行了I/0操作

    • 假设上面是读文件描述表，谁有变化了，把他的位置置为1(操作的是二进制位)

0.3 select()工作过程分析

  • ![[eea064ddbabb2555af7c152c9bba24c5.png]]
  • #面经webserver
  • 首先监听ABCD，将用户态中3,4,100,101的标志为置为1，从用户态拷贝到内核态一份一模一样的
    然后内核监听到了，AB发送了数据，CD没有发送数据，将内核态的的对应标志位3,4置为1，100,101 置为0
    最后在从内核态拷贝回用户态使其保持一致
