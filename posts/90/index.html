1: 学习方法

  • 只关注零散的技术点，没有建立起一套完整的知识框架，缺乏系统观，但是，系统观其实是至关重要的
    • Redis 长尾延迟例子：对 Redis 的线程模型做了分析，我发现，对于单线程的 Redis 而言，任何阻塞性操作都会导致长尾延迟的产生。接着，我又开始寻找可能导致阻塞的关键因素，一开始想到的是网络阻塞，但随着对 Redis 网络框架的理解，我知道 Redis 网络 IO 使用了 IO 复用机制，并不会阻塞在单个客户端上；再后来，我又把目光转向了键值对数据结构、持久化机制下的 fork 调用、主从库同步时的AOF 重写，以及缓冲区溢出等多个方面
  • 在应用维度上，按照两种方式学习: “应用场景驱动”和“典型案例驱动”，一个是“面”的梳理，一个是“点”的掌握。
    • 重点解读一些对 Redis的“三高”特性影响较大的使用案例，例如，多家大厂在万亿级访问量和万亿级数据量的情况下对 Redis 的深度优化
  • 在学习和使用的过程中，可以根据自己的方式，完善问题画像图，把自己实践或掌握到的新知识点，按照“问题 --> 主线 --> 技术点”的方式梳理出来，放到这张图上。这样一来，积累越来越多，画像也会越来越丰富

9: 薪资问题

“你好，HR小姐姐，我还真没考虑过薪资的问题，因为按照我的认识和理解，我认为自己的第一份工作是想学到东西的，因此我更看重公司的工作氛围，特别是技术氛围这一块，我很希望自己能在贵公司学到一些新技术或者一些能够让自己有所成长的东西，我是一个很喜欢技术的人，能够学到让我在学校里学不到的东西是我最看重的，所以薪资这块我确实不怎么在意，差不多就行。 如果您需要一个比较具体的数值的话，我也确实报不出来，我只是觉得自己的水平应该还不至于很差，所以只要自己的薪资能够达到贵公司这次招收的应届生的薪资范围的中等水平就好，如果是垫底的水平我可能不会接受，因为我对自己的技术水平有信心，也相信自己会比绝大多数同批次进入公司的同学表现要好，不管是工作产出上还是与同事相处上，我都对自己很有信心，所以薪资这块，您给个中等水平的offer就好。”

我觉得职位和薪酬都是你的到达那个水平，公司自然会提供给你的。鹅厂有完善的职位薪酬体系，并且对员工的考核也是全方位的，我相信在职的各位前辈必然有他的过人之处，我觉得我企业是能提供我公平完善的考核及晋升机制的。并且我觉得，在第一份工作的选择中，我们更应该考虑的是如何扎根在这里，成长起来，承担自己的责任，并且为企业获取效益，从而让自己更加有影响力。

10: 你最看重一个公司哪三个点

第一看公司的规模、技术积累以及未来发展趋势。

第二看公司的文化和氛围，给员工的工资水平待遇，再好的公司没有这一点也是枉然。

第三看个人在这个公司的晋升和发展 这三点我认为是比较重要的

11: 3~5年内，你有什么职业规划嘛？

作为应届生加入公司，我相信一定有许多东西需要去进一步的学习，希望能在这个大平台，能够相周围优秀的同事们学习，学习公司的文化，通过追赶他们，让自己成为一个更加职业化，专业化的人，我觉得这是一个难得的机会；3~5年内，我的目标是在工作方面超额完成，让自己在工作领域有所建树；在行业方面，希望自己以后能成为领域行家，根据公司脉络发展进步，能够让自己与公司共同进步，实现双赢。

一定要围绕技术这块来规划，根据你的岗位，一年 技术达到什么程度，三年在某个技术领域有深入研究，五年成为技术专家之类的

通过简单了解。知道咱们公司的主要产品有XXXX，咱们现有媒介平台是通过XXX，渠道合作方面包括XXXX

13: 对你面试的岗位的理解

自己对着jd和自己的理解，来形成的一套说辞，我一般会分点来进行递增的理解介绍，比如测试开发，就会从测试、再到测试开发来介绍

首先我选择投递岗位是软件工程师（C/C方向），这主要是因为我个人认为自己具有较好的一个C开发编程基础，结合我简历上的三个项目对软件开发的整体流程都相对比较熟悉，理解操作系统，QT界面设计这块都具有一定的理解，而且本身对算法的落地更感兴趣，并且本身在项目当中也是负责实现的，对软件开发部分会更加熟悉

14: 你觉得在团队中最重要的几个点是什么呢？

执行规划+确认计划+定期跟进沟通进度

15: 可以来实习吗？

如果是签了三方再去实习，现在的导师是不允许去实习的，但是今年年末之后，如果论文和三方都已经签订了，我也拿到了公司的意向书和完成的三方协议，我可以拿着这个和导师说明情况。我也很希望可以提前进入公司提前适应节奏和学习新知识。

16: 你对加班的看法

首先我前些年因为是应届生进入公司，我需要更多的时间来进行学习和完成社会上角色的转换，我觉得公司里也有很多技术需要我去学习。在之后我觉得工作时间是和我的职责和项目进度来保证的，我会尽最大的努力在规定时间内完成工作任务，如果是未完成，或者是紧急工作，我也会适应公司的节奏和加班的强度。

17: 你在团队中担任什么样的角色？

我一般会回答执行者、沟通者、领导者，这三个选一个结合具体的例子来进行解释

18: 如果团队中有消极怠工的成员你会怎么办？

个人是分程度和沟通程度来讲。

首先进行谈心和强调重要性，提高积极度。。。。一步一步延伸，如果还是拒绝合作和消极怠工，那就上级反应情况

19: 实际过程遇到的困难比想象的大，怎么解决

这个…及时向领导反映,说明其真实情况,如果领导说可以延长那你把大概完成时间向领导汇报,但领导说不能延长时间,那你就请求领导增加人手,确保在要求时间内完成

20: 如果公司录用你，你将怎样开展工作？

“首先听取领导的指示和要求，然后就有关情况进行了解和熟悉，接下来制定一份近期的工作计划并报领导批准，最后根据计划开展工作。

关于岗位的问题，关于公司的产品问题，关于公司的项目问题
