KMP 算法理论篇

本文最后更新于：2022年8月17日 下午

题目: 要在文本串：aabaabaafa 中查找是否出现过一个模式串：aabaaf。

暴力匹配: for 循环发现不匹配, 则从头匹配.
暴力的解法显而易见是O(n × m)
^zyqb9e

KMP: 使用前缀表, 就不会从头匹配，而是从上次已经匹配的内容开始匹配，找到了模式串中第三个字符b继续开始匹配。

1 时间复杂度分析

其中n为文本串长度，m为模式串长度，因为在匹配的过程中，根据前缀表不断调整匹配的位置，可以看出匹配的过程是O(n)，之前还要单独生成next数组，时间复杂度是O(m)。所以整个KMP算法的时间复杂度是O(n+m)的。暴力的解法显而易见是O(n × m)，所以KMP在字符串匹配中极大的提高的搜索的效率。
为了和力扣题目28.实现strStr保持一致，方便大家理解，以下文章统称haystack为文本串, needle为模式串。

都知道使用KMP算法，一定要构造next数组

2 构造 next 数组

构造next数组其实就是计算模式串s，前缀表的过程。 主要有如下三步：

  1. 初始化
  2. 处理前后缀不相同的情况
  3. 处理前后缀相同的情况
    循环不变量: 遇见冲突看前一位的 next 数值, 然后回退到下标为 next 数值的位置继续比较
    ![[KMP 算法理论篇 2022-08-16 17.16.15.excalidraw]]

注意next数组是新前缀表（旧前缀表统一减一了）。

背住过程

  • 起始时，next[0]=0，j从0位置开始，i从1位置开始

  • 如果s[i] != s[j] ：将j指针指向前一位置的 next 数组所对应的值，即j = next[j - 1]
    这个过程是循环进行，直到s[i] == s[j] 或者 j=0
    如果j == 0, s[i] 和 s[j] 还是不相等，则next[i] =0，i往后移动, j 不变

  • 如果s[i] == s[j]：next[i] =j+1; i和j同时后移（这个过程一直同样循环进行）

总结
KMP 算法的应用范围要比 Manacher 算法广，Manacher 算法只能应用于「回文串」问题，相对比较局限，而「子串匹配」问题还是十分常见的。

背过这样的算法的意义在于：相当于大脑里有了一个时间复杂度为 O(n)的 api 可以使用，这个 api 传入一个原串和匹配串，返回匹配串在原串的位置。

因此，三叶十分建议大家在「理解 KMP」的基础上，对模板进行背过 ~
