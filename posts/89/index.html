秋招经验总结贴的副本

本文最后更新于：2023年1月4日 上午

📝秋招经验总结贴的副本::

录友们好，我是Woods Lin，感谢卡哥的邀请，给大家整理一波经验贴，我的秋招也是终于迎来了尾声，希望我的经历对大家有所帮助

1: 个人情况及经验总结

本人中流985机械硕211本，从21年10月开始就准备秋招事宜，前期对岗位的认清不够清楚，直接一股脑冲CV算法岗了，报了几个课程班信心满满的冲算法岗，这是当初的准备情况介绍，首次提问。然后在春招找实习的过程中被大打击，认清了自己的水平和能力与算法岗有挺大的差距，也就有了这个提问：算法岗失利，然后开始全力准备C++开发岗了（虽然最终还是面上了华为的AI工程师岗位，但是自己已经对算法岗心如死灰再加华为实习与AI没有任何关系，因此在实习期间还是继续准备开发岗）

其实综合来看自己的今年属实是运气好了，技术上我的劣势还是很大，和星球里很多大佬比不了…也就靠着学历勉强蹭了很多面试，边面边成长，最终也算是拿到了一些offer，虽然大部分都是大白菜，但是今年这个形势也不要求更多了。希望我的学习总结对大家有点参考把，以下是各个方面的学习总结

2: 算法题

这块由于自己是非科班，所以早早开始了刷题，大概是从21年的10月份开始每天留些时间刷刷题了；

  • 一开始并不知道代码随想录，就在力扣上乱刷，每天刷两道题一早上就过去了，但是常常都是看着答案才做出来的；直到22年1月份从同学那里知道了代码随想录，开始跟着进行了刷题，到正式秋招随想录应该是完整的刷完了3遍，可以说是我的算法指明灯了，系统的学习了各个方向的解题思路，奠定了比较好的基础。
  • 其余的算法题资料包括力扣上的各种leetbook，学习计划，每日一题以及有空就会打打周赛看看自己对新题的感觉是怎么样的。
  • 当然还有一个剑指offer
  • 以及自己报的一个算法课程，kkb的门徒计划（被坑钱了就不推荐），到时候把题单整理一下

到目前为止自己在力扣也是刷了1100+道题目，还是蛮有成就感的一件事情哈哈哈。直到现在我也还是保持着每天的刷题习惯，作为自己每日的小任务。一方面是自己感觉做题有一定的成就感，其次是想说保持手感提高自己的代码能力把。

所以这块的建议就是，越早开始越好，算法题真的没办法速成，一定要跟着随想录把算法思维构造好，感觉基本的面试题把随想录刷够3遍每道题做透就是足够的了。

3: C++

C这块自己其实也算是没啥基础，只是本科学习过相关课程，懂一些基本的语法，然后4月份卡哥建议我转C开发后就开始按照卡哥的路线进行学习了；

  • 4月份之前的进度大概是C++ Primer看了前8章，后续就草草的把Primer过完了，因为Primer内容太多了，我目前对他几乎没有什么印象哈哈哈，更多是作为工具书
  • 侯捷老师的视频当初也是快速过了两遍，并对应的整理的一点笔记，关于整理笔记这块我还是蛮推荐大家边看边整理的，每次复习都是不同的感受。（不过侯捷老师的视频我也没有完全吃透，算是了解了点C++的一些面试可能涉及到的点，而不是只会用来写算法题把
  • Effective c++和Effective STL建议好好看两遍！我直到最近才在看第二遍，看了真的感觉很多是面试会问到的点，同建议自己整理下
  • 其余的面试资料就是阿秀的学习笔记了，秀哥把很多面试题都整理下来了，对应刷了3遍，感觉面试的题几乎都有了；另外还看了力扣的C++面试突破，感觉也确实不错，面试够用了。

其实我对于C++的准备是比较不充份的，因为4月才开始转码，中间还去实习了，所以这块建议大家尽早开始，整理好笔记，后续看八股文问题会感觉很熟悉；我之后也会把我整理的笔记发到星球上~

4: 计算机基础

计算机基础主要就分成四大件了，操作系统、计算机网络、MySQL、Redis，这块我也是在卡哥建议我转开发后才开始学习的，十分仓促，但是花的时间还是比较多，最终面试的问题也都答得还可以~下面分块讲一下

  • 操作系统：
    • 首先是看了王道的操作系统视频（这个好像是在准备算法岗的时候就有看的，4月份之前看完了一边，也整理的对应笔记，虽然后续没有看），这个视频还是比较不错，对整体有了初步的印象，也有助于后面的各种文章阅读；建议0基础的录友可以快速过一遍，充足时间的话尽量都理解了。
    • 然后4月份开始看小林图解系统了，一开始看小林真的痛苦，因为基础比较差，一篇文章一般都要看1个多小时（加整理自己的笔记）但是真的感觉小林这个系列讲得真的很好，强烈推荐大家好好钻研；我总共是整体看了3遍（1遍快速看有印象，一遍ipad笔记涂涂画画，一边整理面试可能问到的点了），每次看都有不同的体会，只能说这块把小林吃透了，面试肯定也没问题了。
    • 其余的资料就是阿秀的学习笔记了，可以当八股文来背，自己也整理了下
  • 计算机网络
    • 首先是看了b站的计算机网络微课堂（都是卡哥学习路线里推荐的），这个视频我也觉得是十分好，一下子对计网有了初步的概念，同样是为了后续看小林图解提供了基础
    • 接下来就一样是看小林的图解网络了，这部分小林真的讲得很好，强烈推荐，同样是看了3遍整理自己的笔记
    • 其他的资料同样是阿秀的学习笔记，在前面的基础下看这个整理的面试题，又是加深了一边印象，进一步理解
    • 另外自己最开始学习还看了卡哥推荐的一些书籍（图解HTTP，图解TCP/IP），但是当初时间比较赶，有点囫囵吞枣的感觉，看完后并没有太深的印象，所以我建议和我一样0基础的同学先看视频，有了理解之后可以再看书
  • MySQL
    • 这部分的学习我可以说是拖得很晚了，最开始一样是跟着卡哥的学习路线看了哈工大数据库系统，但是同样看得有点痛苦。。记不下来，所以最后放弃这个视频了，时间也不允许我慢慢看视频了
    • 然后就依照前面学习操作系统和计网的经验，直接上小林，这部分看得更痛苦了，非科班的我对这块完全没有基础，但还是硬啃下来了，最终复习完3遍之后也有了自己的理解
    • 另外还有一个对我很有帮助的资料就是MySQL45讲了，真的讲得很细致，涉及了很多面试的问题，目前我也在看第二遍整理笔记当中。
    • 其他的资料同样包括了阿秀的学习笔记，同样看了2遍多，加深印象。
  • Redis
    • Redis一开始也是看了卡哥推荐的尚硅谷的视频，但是感觉讲的有点浅，大概过了一遍没有过于纠结细节
    • 重点仍然放在了小林的图解Redis上，这块小林的整理会少一些，看完MySQL后感觉还是比较好理解的，同样是看3遍整理自己的笔记
    • 另外的学习资料是极客专栏的Redis核心技术与实战，讲得也很好，很有助于理解，目前也是在看第二遍整理笔记中
    • 最后同样是阿秀的学习笔记

计算机基础对我来说是很难啃的，我基本都是按照视频->小林图解系列->极客专栏->阿秀学习笔记 这样子学习下来的。一层一层的不断巩固学习形成了自己的知识体系，也很好的应对了面试问题，但是距离真正掌握仍然是有很长的距离。

5: 项目

我的简历上放了3个项目，一个华为的实习（偏向图像处理，和开发关系不大），烂大街的Web服务器以及研究生的课题项目（C++开发的图像处理算法，同样关系不大 ）。所以项目这块其实是我的劣势，这也是为什么我一直在注重于计算机基础的巩固的原因。

这块我的建议其实是自己至少做两个项目，如果要投递后端开发的录友最好涉及一下MySQL和Redis，把面试官往自己熟悉的方向引，这样子面试就很自如了（当然只是自己在后悔没有足够的时间把项目做好，大家自己结合情况考虑把）。我因为就是只有一个简单的Web服务器可以问，所以每次我都会把面试官引向计网底层原理的挖掘，其余的MySQL和Redis其实我被问到得很少。

6: 秋招面试结果

总共投递了近80家公司，算是小海投把哈哈哈，由于实习+学历的缘故，基本都会给个笔试，也是比较多公司给我进行了面试（总共面了50多场）。在一次次面试当中得到了成长，最终也顺利拿到了一些offer，以下给大家整理一下拿到的几个offer及其对应的面经

字节客户端开发工程师（小sp）

字节最开始投的是抖音客户端，可惜二面算法题没做出来，挂了，然后番茄小说捞了我，也没想到自己能拿到，属实是运气爆棚了，一下是番茄小说的3面整理

一面 60min

  • 自我介绍

  • 说一下你了解的数据结构

  • 讲一下哈希表内部的原理

  • 怎么处理哈希冲突的

  • 怎么判断一个单链表是否有环

  • 怎么判断两个链表是否相交，怎么确认交点呢

  • 怎么用栈实现队列的功能

  • 再说一下用户态和内核态，他们之间怎么切换的

  • 你了解哪些中断呢？（回答了硬中断和软中断，不太对感觉）

  • 介绍一下父子进程

  • 什么是虚拟地址什么是物理地址

  • 介绍一下缺页中断

  • 如果页表满了怎么办，你了解什么置换算法

  • 再介绍一下动态链接和静态链接的区别

  • 动态链接和静态链接是在什么阶段执行的呢，编译?执行？调用？

  • 静态变量和全局变量的区别（没说出具体的区别）

  • 说一下数据库事务

  • 你知道抓包吗？网络过程间的状态查询

  • 可以抓HTTPS的包吗？

  • 那我们作为开发者如何防止抓包呢？除了君子协议外（不懂，要防止攻击者）

  • 程序输出分析，一个数组指针问题，一个sizeof数组元素的分析，一个位运算计算二进制有几个1

  • 算法题：目前的链表奇数位置升序偶数位置降序，怎么对链表进行排序

    采用了拆分链表然后翻转偶数链表然后合并有序链表的思路，因为拆分后忘记断开导致程序输出的时候死循环了哈哈哈很尴尬，和面试官找了半天

  • 再介绍一下智能指针把（因为我使用虚拟头但是没有delete，程序有警告哈哈哈）

  • 智能指针有哪几种，都有引用计数吗

  • 再介绍下进程间有哪些通信方式

  • 对称加密和非对称加密介绍

  • 反问

二面 45min(以为又要挂了竟然没有)

  • 自我介绍
  • 介绍一下TCP/IP的四层模型
  • 假设浏览器访问网址，这四层分别会发生什么事情呢
  • 网络层会涉及到什么协议呢（ARP）
  • 网络IP地址转化流程，外网IP和内网IP是怎么转换的
  • 比如说平时用路由器，多个手机连接同个路由器，那手机获取到的地址是什么地址呢
  • 路由器是否会经过内网和外网的IP转换呢
  • 这个IP转换是怎么实现的，主要通过端口号是吗
  • 假设手机a和手机b都在同一个wifi下，想要手机a通过wifi向手机b发生一个文件，你觉得在网络上面可以用什么协议做这个事情
  • 比如说HTTP协议可以做嘛？（我回答没必要，不需要应用层的传输，在网络层就可以了，但是其中的协议不太了解）
  • 那TCP协议可以处理嘛？（要传输肯定也是可以的）
  • 那为什么说HTTP不可以（。。我不是说不可以，就是说可以不用，使用HTTP也是可以交互的）
  • 假设现在有两个路由器，手机a只能连接路由器a，手机b只能连接路由器b，那路由器a和b要如何配置呢，从网络原理如何分析呢（回答通过路由器表配置mac地址啥的，不会，乱说）
  • 那假设现在手机a ping一下手机b会发生什么事情？他的IP传输过程是怎么样的（ICMP）
  • 动态规划、贪心和搜索这三个的区别
  • 那你觉得动态规划最关键的要素是什么（子问题，相互独立）
  • 那贪心有什么特点呢
  • 搜索的算法有哪些呢
  • 深度和宽度分别解决什么问题呢
  • 搜索的剪枝是什么呢
  • 字典树是怎么样的一个数据结构呢
  • 字典树构造的复杂度是多少
  • 三个算法题
    • 第一道题目是给你一个字符串，可以讲任意一个字符转为任意一个小写字符，这个操作可有m次，问转化后的字符串中最长的相等子串长度，比如"abcdac" ，2次操作，可以转化为"abcccc" ，最长是4，返回4（我思路不太对，用的滑动窗口做，没做出来）
    • 第二个题目感觉我有点菜送了个分，实现二进制字符串的相加，比如"10001" + “1101” = “11110”
    • 第三个题目还蛮有意思，给你一个数字n，有两种操作，（n + 1）% 1024, （n * 2） % 1024，求将n转化为0的最小操作次数（讲了一下用二进制的思想求解，+1就是+1，*2变成左移，转为1024的最少次数；然后让我想一下dp是否能做，问子问题怎么定义）

三面 60min

  • 自我介绍
  • 研究生项目介绍，问了很多相关问题，如何解决之类的
  • 平时C用的比较多是吧，介绍一下一个C代码从源文件到可执行文件的过程
  • 编译这个步骤都做什么工作了解吗
  • 做一个设计题吧，设计一个类似力扣的网站考察系统，支持提交代码，屏幕端运行后输出结果，可能产生很多种结果，比如编译错误，运行错误，运行超时，运行超内存，主要就是介绍如何设计代码，如何在服务端将代码编译后运行将结果传递给客户端，主要就是介绍每个模块怎么设计（捕捉各个信号，传递给指定的处理模块，利用定时器记录是否超时以及记录内存空间）
  • 代码怎么提交到服务器呢，选用什么通信协议呢（HTTPS）
  • 选用GET还是POST呢（POST吧）
  • 为什么GET不行呢，如果抛开语义是否可以呢（应该可以的）
  • 是否有什么场景GET无法请求呢（URL超长传递大小限制）
  • 定时器你这边怎么设计呢（维护定时器的堆）
  • 死循环的话你怎么计算时间呢（主线程一直循环处理事件，看是否超时）
  • 为什么你使用两个线程而不是进程呢？（进程通信开销大）
  • 撇开进程通信的开销的话，进程通信有什么优势嘛（空间相互独立）
  • 针对堆内存和栈内存你怎么去获取呢？（堆的话重载new，将开辟的内存空间放置在一个链表上，栈的话利用系统查看）
  • 记录到链表上的话也需要新建一个节点，那不会导致无线递归调用new嘛？（按照类进行重载new）
  • 万一说别人的代码里面新创了一个类你也需要针对该类进行重载new了？
  • 那你记录的这个链表还需要记录时哪个类开辟的？
  • 设计模式你有了解吗
  • 用C++实现一个单例模式吧
  • static对象第一次初始化在什么时候呢（懒汉模式，判断是否初始调用，是的话利用new开辟）
  • 多线程下是不是有问题（加锁，或者弄成饿汉模式）
  • 饿汉模式下是否就不是第一次调用的时候开辟了
  • 平时写代码的话有用到这些设计模式吗
  • 算法题，深拷贝一个带random指针的链表
  • 反问，工作氛围，学习建议

同花顺 C++开发工程师—B2B （23 * 15）

一面 52min

  • 自我介绍

  • 公司大介绍，balabala，主要业务方向场景等等，然后：我看你的研究生项目还有华为实习经历都是偏硬的C++开发，想问下你的职业规划或者说倾向是什么？（这块可能说了快20min。。）

  • 聊聊技术问题把，你怎么看待C和C++，介绍一下差别

  • 你说到多态，多态是什么，怎么实现的

  • 虚函数表怎么起作用的？

  • 能继续介绍下封装吗？

  • 继承也介绍一下

  • 多继承了解吗

  • 菱形继承了解吗

  • 如果不使用virtual，可以访问指定B类或者C类的成员吗

  • 网络编程你了解到什么程度？是调用接口还是？

  • 那你平时学习一般都从哪里学，问题搜索其实很重要

  • 你这个web服务器用到了登录，是用什么数据库？

  • MySQL你了解到什么程度

  • 编程题，二分查找第一个出现target的位置

  • 反问，部门具体业务和技术栈，继续大介绍

二面 35min

  • 自我介绍

  • 最近在干什么（学习，补充自己不足的地方）

  • 学什么，哪里比较不足（MySQL，Redis以及各类实际业务的场景问题）

7: 总结（碎碎念）

总得来说，今年的秋招确实不容易，我也是靠着学历+实习获得了比较多的面试机会，自己也深知技术能力和很多录友还有着较大的差距，也是在不断查缺补漏慢慢成长，接下来的每天我仍然会坚持每日打卡，记录自己的学习。我的经历或许不适合所有人，但还是有几点建议想和大家分享一下

  • 非科班冲算法岗的小伙伴真的要有心理准备，道阻且长，顶会+竞赛+学历+项目都有的可以一冲；千万不要像我一样没有了解清楚盲目冲锋，浪费了自己很多时间。如果真的对算法岗十分感兴趣，建议多从竞赛和项目下手，多去打打kaggle，还是有机会的；

  • 刷题一定尽早，养成解题思维很重要，每日保持手感也很重要，在面试中多和面试官说说解题思路（从数据规模，题目特征等进行切入）

  • 计算机的基础很重要，可以深挖一块，对基础的理解也很有助于项目的总结，放在简历上在面试中就可以引导面试官到那个方向；另外八股文建议大家整理一套自己的面经，按方向进行划分，把每次面试中遇到的问题以及网上的各种面经分类整理，不断面试不断成长。

  • 最后就是建议大家善于规划，制定好每日计划，多多复盘多多总结，充实每一天

我的秋招终于要画上了句号啦，最近也会把自己秋招的相关资料都整理一下，祝大家都能拿到心仪的offer~一起加油！
